Sphere Cast

I have found in my time making Beer Run that it is vital to ensure coins and obstacles were not overlapping with each other as it caused frustration when you died while collecting coins. I will not be making a tutorial for Sphere Casting as I will in stead go through and break down the process I went through to implement Sphere Casting into Beer Run.

I'll start by explaining how my spawning works. The obstacles in the game are on timers and spawn at random distances from the player, the coins are the same only they spawn a set distance away to keep them consistant. Now because the coins were spawning after obstacles I had to do a more complex Sphere Cast to get this to work. To ensure that not a sinlge one of the 8 coins per group were not colliding with an obstacle I had to do 8 sphere casts in a line and if any of them returned a collision a new lane needed to be selected and and another search conducted.

Ok lets start at the very beginning of this function. First of all I've got a for loop set up to search 5 times, one for each lane, if a lane is found before all 5 iterations then the function will simply be returned.

The hitcount variable is used later to determine amount of successful iterations. The First thing done inside the for loop is selecting a random lane position.

The next step in this process is determining the exact positions the coins would sit in this current lane. We need this to sphere cast and determine any collisions.


Now that the position data is set up and stored its time to do the Sphere Cast. First of all I have a for loop set for 8 iterations. If the Sphere Cast is returning no collisions the for loop will continue otherwise it will break out and start the function again and select a new lane. In the below code snippet you will see the first thing I create is a "var collision". This is a local variable that is set the the Sphere Cast. Sphere Cast returns a list of collisions that are made on any particular position. To complete a Sphere Cast you'll need to pass in the correct parameters. It takes in a position, a radius for the search and then it takes in which layer of objects in the world you would like to search. The underlined line of code is the Sphere Cast.


You will see above I use the collision variable to check if a collision was made, if it was the loop needs to be broken and another lane needs to be selected, otherwise the loop will continue until either a collision is found or until all 8 iterations are complete meaning the coins can be placed in the correct spot. I use the hit count variable to keep track of how many successful iterations there has been.


Below you will see the final part of this function. If the hitcount hits 8 and there is no collision it will populate a list preparing the coins to be placed in the world. The position of each coin is then set and then placed in the world. The function will return and clear all lists is preparation for the next group of coins


With this function I can successfully place coins in their own lane and ensure there are no obstacles in the middle of them. Learning the Sphere Cast function was incredibly helpful and has noticeably benefited game play, it is an amazing function to know if you are using the Unity Engine, it could be helpful in many different instances.
